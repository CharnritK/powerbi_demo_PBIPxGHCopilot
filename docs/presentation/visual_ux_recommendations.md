# Visual and UX Recommendations

## Notebook output style
- Keep each notebook section short and purpose-driven.
- Show small explanatory markdown before each code cell.
- Prefer clean DataFrame tables over dense JSON.
- Add only one simple chart after the DAX result.
- Avoid long scrolling outputs.

## Folder structure slide
- Show the repo tree as a left-aligned code block.
- Highlight only 5–6 important folders with callouts:
  - notebooks
  - scripts
  - pbip
  - data
  - config
  - docs

## Screenshots or mockups to include
- Notebook auth selection cell
- Workspace list output
- Dataset list output
- DAX query result table
- PBIP folder concept slide
- One architecture flow diagram

## Trust-building design choices
- Use consistent naming across repo, notebook, and slides
- Avoid overly clever folder names
- Keep visuals white-space heavy
- Use no more than one accent color plus neutral tones
- Show limitations explicitly instead of hiding them

## Minimalist visual direction
- White or very light background
- One dark neutral text color
- One blue accent for highlights
- Rounded screenshot containers
- Simple icons for notebook, API, semantic model, and user/app identity

## Presenter UX
- Open only one main window at a time
- Keep a pre-filled `.env` off screen
- Preselect workspace and dataset for the live demo
- Have a backup static screenshot for each notebook step
